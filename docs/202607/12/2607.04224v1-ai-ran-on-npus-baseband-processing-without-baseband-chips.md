## 摘要
AI-RAN旨在统一人工智能和无线接入网络在共享计算平台上的工作负载。虽然这一范式目前主要是在图形处理单元（GPU）上展示，但尚不清楚专为推理优化的AI加速器——神经处理单元（NPU）是否也能支持无线基带处理。本文首次通过解决基带工作负载与NPU架构之间的根本性不匹配问题，给出了肯定答案。计算同构性确实存在：NPU专用于推理的矩阵和向量引擎天然涵盖物理层操作。然而，NPU架构本质上是为密集张量AI推理而设计的，而非基带处理。这种架构不匹配表现为不同的优化目标：传统基带处理最小化算术运算，而NPU性能则要求最大化引擎利用率。我们通过将通信算法重构为AI计算基元来弥合这一差距，优先考虑引擎利用率而非算术运算次数。我们在Ascend 310B1边缘NPU上使用完整的OFDM收发器验证了这一点，并通过USRP X300在3.0 GHz频率下演示了端到端的空中传输。

## Abstract
AI-RAN aims to unify artificial intelligence and radio access network workloads on a shared compute substrate. While this paradigm has so far been demonstrated primarily on Graphics Processing Units (GPUs), it remains unclear whether Neural Processing Units (NPUs), which are AI accelerators optimized for inference, can also support wireless baseband processing. Here, we provide the first affirmative answer by resolving the fundamental mismatch between baseband workloads and NPU architecture. A computational isomorphism exists: matrix and vector engines NPUs dedicate to inference inherently cover physical-layer operations. Yet NPU architectures are natively shaped for dense-tensor AI inference, not baseband. This architectural mismatch surfaces as opposing optimization objectives: traditional baseband minimizes arithmetic operations, whereas NPU performance demands maximizing engine utilization. We close this gap by reconstructing communication algorithms onto AI compute primitives, prioritizing engine utilization over arithmetic count. We validate this with a complete OFDM transceiver on an Ascend 310B1 edge NPU, demonstrating end-to-end over-the-air transmission via USRP X300 at 3.0 GHz.

## 1. 论文的核心问题与整体含义（研究动机和背景）

AI-RAN（人工智能与无线接入网融合）旨在将AI与RAN工作负载部署在共享计算硬件上。此前相关演示主要基于GPU（如NVIDIA Aerial），但NPU（神经处理单元）——一种面向推理优化的AI加速器——能否支持无线基带处理尚不明确。本文的核心问题是：**NPU能否执行完整的基带信号处理**，从而将AI-RAN的硬件选择范围从集中式GPU服务器扩展到分布式边缘NPU。作者通过识别基带操作与NPU计算引擎之间的**计算同构性**（computational isomorphism），即物理层线性操作可表达为矩阵乘法、非线性操作可表达为元素级矢量运算，首次证明了NPU可以替代专用基带芯片执行完整的OFDM收发链。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
NPU的矩阵引擎（Cube engine）和矢量引擎（Vector engine）是AI推理的核心，也天然涵盖基带处理所需的矩阵-矢量运算。但传统基带算法（如FFT蝴蝶、FIR滤波、LDPC稀疏XOR）是面向最小化算术复杂度设计的，与NPU最大化引擎利用率的要求相悖。因此，作者提出**将基带通信算法重构为AI计算基元**，优先提升引擎利用率，而非减少运算次数。

### 关键技术细节
- **算子形状不匹配的解决**：
  - **线性操作**：恢复为显式矩阵乘法形式，映射到Cube引擎。例如：
    - DFT/IDFT：用cos/sin基分解法替代蝴蝶算法，将整个帧的符号打包成批量矩阵乘法。
    - RRC脉冲成形：从滑动窗口FIR重写为多相Toeplitz矩阵乘法。
    - LDPC编码：从比特XOR提升为int8生成矩阵乘法+模2约减。
    - 子载波提取：重写为置换矩阵乘法。
  - **非线性操作**：重写为纯算术形式（cast、add、multiply、clamp），映射到Vector引擎。例如：
    - QAM调制：用算术链替代查表。
    - CFO补偿：通过角度和恒等式递归生成相位，避免每样本三角函数计算。
- **数据移动瓶颈的解决**：
  - **片上流水线融合**：将相邻算子（如LS信道估计+ZF均衡）合并为一个核，中间数据保留在片上，避免每算子来回访问DDR。
  - 融合还包括：脉冲成形写入FFT布局消除CP去除核；调制直接写入变换输入步长消除主机填充核。
- **GPU对应接口**：作者类比CUDA，将NPU的核级编程接口称为nCUDA（本工作使用AscendC）。

## 6. 论文的主要结论与发现

1. **首次证明**：商用边缘NPU（Ascend 310B1）可以完全替代专用基带芯片，执行完整OFDM收发链，并通过空中链路实现无误码传输（大部分帧零BER）。
2. **性能-功耗优势**：在8 W封装下，NPU实现TX 4.1 帧/秒/瓦，是边缘GPU（0.6 帧/秒/瓦）的6.8倍，是桌面CPU（0.4 帧/秒/瓦）的10倍。
3. **算子瓶颈分布**：NPU在Cube引擎算子（RRC滤波、OFDM IFFT、LDPC编码）上表现最优；GPU在LDPC解码上最快；CPU在标量密集型算子（QAM调制、CFO补偿）上最快。但端到端功耗约束下NPU整体能效最高。
4. **可扩展性**：方法可迁移到高性能NPU平台（更高端的Ascend系列），利用更丰富的硬件原语进一步提升性能。
5. **AI-RAN三支柱**：NPU同时支持AI-for-RAN（替换为神经网络接收机）、AI-and-RAN（共享计算）、AI-on-RAN（边缘AI服务）。

## 8. 不足与局限

- **绝对延迟仍远高于5G NR子毫秒预算**：NPU接收延迟46.2 ms，远不能满足实时要求。作者承认需升级到更高性能NPU平台。
- **实验覆盖有限**：仅测试了SISO链路、固定OFDM参数、无多用户MIMO和干扰场景。未评估不同带宽、码率、调制阶数下的性能。
- **缺少与专用基带芯片（FPGA/DSP）的直接对比**：仅有与GPU/CPU的对比，而实际基带部署常使用FPGA/DSP。NPU的能效与这些传统方案对比如何没有提供。
